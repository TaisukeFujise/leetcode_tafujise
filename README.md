## Goal
*「他人のコードを読んでレビューして、専門家のブレの範囲はこれくらいで自分はその選択肢の中からこれを選択するという感覚を手に入れること」*
- LeetCode medium を初見で10分以内に書き上げれる
- 発想の方法の説明ができ、十分な「連想」と「資料調べ」ができる
- 関連するライブラリの中身も書ける
- 言語のバージョン間の差異、言語仕様か環境依存かなどにも気を配れる

## 進め方
- Step1
  - 5分考えてわからなければ、答えを見る
  - 答えを見て理解したら答えを隠して書く。5分迷ったら答えをみる。
  - 答えを見てしまったら全部消してやり直す。
- Step2
  - 次にコードを読みやすくするようにできるだけ整える。
    - 過去に同じ問題を解いている人の解答を読み、それを踏まえて改善。
    - 以下の点をコメントに残す；
      - 講師陣のコメントに想定されること
      - 他の人のコードを読んで考えたこと (感性を養う)
      - 改善するときに考えたこと
- Step3
  - 全部消して時間を測りながら書くを10分以内に3回連続できるようにする
  - レビュー依頼＋レビューを元に修正
